高级立绘系统详细教学

全面升级的立绘控制方案：方位 → 服装（dress）→ 表情（face）

← 返回主页

概述

高级立绘系统（也称「新系统」）是引擎对旧版简单立绘系统的全面升级。旧系统中角色立绘是一张完整图片（通过 lh1 等资源 ID 引用），而新系统将立绘拆分为三个层级：方位 → 服装（dress）→ 表情（face），每个层级独立配置图片和坐标，实现了服装与表情的自由组合，大大减少了图片资源的制作量。

与旧系统的关系

  • 高级立绘系统与旧版简单立绘系统完全兼容，可在同一条 chars 指令中混合使用。
  • 你可以随时在一个剧本中混用新旧两种立绘——引擎会自动检测指令格式并使用对应的渲染方式。
示例：新旧系统混合
chars="[角色B 右 lh12345], [角色A 中 正面 服装1 表情101]"
       └─ 旧系统 ─┘      └─── 新系统 ───┘

一些建议

  • 不推荐一边改变角色的立绘一边移动其位置
  • 不推荐立绘进场时立刻带上动画（如`[发抖]`等）

一、语法格式

基本语法

chars="[<角色标识> <方位> <服装标识> <表情标识> <可选修饰词...>]"
  • <角色标识>：必须在 CHAR_DIFF_CONFIG 中配置，且必须紧跟在 [ 右边（无空格）。
  • <方位>：决定使用哪一组 dress/face 配置，如 正面、侧面、左侧面 等。此名称完全由你在配置中自定义。
  • <服装标识>：对应配置中 dress 数组内某个对象的 id。
  • <表情标识>：对应配置中 face 数组内某个对象的 id。
  • <可选修饰词>：位置、动作、渐变、亮度等（详见旧系统文档 illustration.md）。
注意：百分比缩放指令（如 [30%]）和包含缩放效果的动画（后退/retreat、前进/forward、吓一跳/scare）在新版立绘中会被自动屏蔽，详见第六章。

基础示例

chars="[角色A 中 正面 服装1 表情101]"

其中：

  • 角色A — 角色标识符
  • 中 — 水平位置修饰词（在画面中央）
  • 正面 — 方位标识符（使用正面朝向的 dress/face 配置）
  • 服装1 — 服装差分 ID
  • 表情101 — 表情差分 ID

连续序列指令

多个状态可用逗号分隔，引擎会按 500ms 间隔自动逐步切换，类似于关键帧动画：

chars="[角色A 左 正面 服装1 表情101, 中 正面 服装1 表情105, 右 正面 服装1 表情110]"

序列特性：

  • 每步间隔 500ms，自动从第一步执行到最后一步。
  • 在序列进行中点击鼠标，角色会瞬间跳到最终状态（最后一段所描述的位置和差分）。
  • 可在任意段前插入独立的 瞬 / instant 标记，表示该步瞬时切换，不使用过渡动画：
chars="[角色A 左 正面 服装1 表情101, 瞬, 中 正面 服装1 表情105]"
                                                        ↑ 这一步瞬间完成

二、配置文件：illustration.js

2.1 CHAR_DIFF_CONFIG — 立绘差分配置

这是新系统的核心配置，位于 illustration.js 中。结构如下：

window.CHAR_DIFF_CONFIG = {
    "角色名称": {
        "方位名称": {
            dress: [
                { id: "服装ID", file: "文件名", x: 50, y: 0, z: 0 }
            ],
            face: [
                { id: "表情ID", file: "文件名", x: 50, y: 50, z: -50 }
            ]
        }
    }
};
重要：配置变量必须使用 window. 前缀定义，否则引擎无法访问。

完整配置示例：

window.CHAR_DIFF_CONFIG = {
    "角色A": {
        "正面": {
            dress: [
                { id: "服装1", file: "charA_front_dress1", x: 50, y: 0, z: 0 },
                { id: "服装2", file: "charA_front_dress2", x: 50, y: 0, z: 0 }
            ],
            face: [
                { id: "表情1", file: "charA_front_face1", x: 50, y: 50, z: -50 },
                { id: "表情2", file: "charA_front_face2", x: 50, y: 50, z: -50 }
            ]
        },
        "侧面": {
            dress: [
                { id: "服装1", file: "charA_side_dress1", x: 33.85, y: 0, z: 0 }
            ],
            face: [
                { id: "表情1", file: "charA_side_face1", x: 31.61, y: 33.21, z: 0 }
            ]
        }
    },
    "角色B": {
        "正面": {
            dress: [
                { id: "服装1", file: "charB_front_dress1", x: 50, y: 0, z: 0 }
            ],
            face: [
                { id: "表情1", file: "charB_front_face1", x: 50, y: 50, z: -50 }
            ]
        }
    }
};

配置规则：

  • 所有标识符（角色名、方位名、服装ID、表情ID）均可自由命名——支持中文、英文、日文，甚至是任意自定义字符串，只要能正确识别即可。
  • 同一个 dress 或 face 数组内，id 不可重复；跨数组之间可以重复。
  • file 字段建议直接写文件名（如 "char_front_dress1"），无需完整路径；也支持完整路径。
  • 每个方位下可以配置任意数量的服装和表情差分，无数量限制。

2.2 CHAR_PATH_MAP — 子文件夹路径映射（可选）

当你的立绘资源存放在特定子文件夹中时，需要配置此项。如果不使用子文件夹则可忽略。

window.CHAR_PATH_MAP = {
    "角色A": "/assets/chars/角色A",
    "角色B": "/assets/chars/角色B"
};
重要警告：CHAR_PATH_MAP 中配置的路径所对应的文件夹必须实际存在。

如果你在 CHAR_PATH_MAP 中声明了某个角色的映射路径（例如 "角色B": "/assets/chars/角色B"），但该文件夹在磁盘上不存在或已被删除，则该角色所有的立绘（dress 和 face）将无法显示——引擎会使用映射路径拼接最终 URL，浏览器请求该 URL 时得到 404。

示例：若配置了 "角色B": "/assets/chars/角色B"，但 /assets/chars/角色B/ 文件夹不存在，则使用 [角色B 中 正面 服装1 表情101] 时角色B 的立绘将全部加载失败。

路径解析内部机制

引擎的 resolveCharFilePath 函数在查找立绘文件时，按四级优先级解析路径：

优先级规则说明
1完整路径直接返回若 file 字段本身包含 / 或 \，视为已提供完整路径
2命名空间缓存命中引擎首次调用时预填充 CHAR_FILE_MAP，key 格式为 角色名_fileID（如 "角色A_charA_front_dress1"），命中后直接返回预填充的完整路径
3CHAR_PATH_MAP 显式映射若 CHAR_PATH_MAP 中有该角色的映射 → 拼接 {映射路径}/{file}.png
4默认回退均未命中时 → 回退至 /assets/chars/{角色名}/{file}.png

命名空间隔离

引擎预填充 CHAR_FILE_MAP 缓存时，key 统一使用 角色名_fileID 格式，确保每个角色的查找完全独立，互不干扰。即使多个角色（如「角色A」和「角色B」）在 CHAR_DIFF_CONFIG 中使用了相同的 file 名，也能各自正确命中自己的资源路径。

CHAR_FILE_MAP["角色A_charA_front_dress1"]  → 指向角色A的资源路径
CHAR_FILE_MAP["角色B_charA_front_dress1"]  → 指向角色B的资源路径（独立）
"角色A" 查找 → 命中 "角色A_charA_front_dress1" → 正确

2.3 CHAR_DIFF_OFFSET — 表情位置校正（可选）

用于微调剧本中表情相对于身体立绘的位置，解决可视化窗口与剧本显示不一致的问题。

window.CHAR_DIFF_OFFSET = {
    "全局": {
        x: 0,  // 全局 x 轴校正偏移（只影响表情层）
        y: 0   // 全局 y 轴校正偏移（只影响表情层）
    },
    "角色A": {
        "正面": { x: 0, y: 0 },
        "侧面": { x: -0.5, y: 0 }
    },
    "角色B": {
        "正面": { x: 0, y: 0 },
        "侧面": { x: 0, y: 0 }
    }
};

叠加规则：

  • 计算公式：最终偏移 = 全局偏移 + 角色偏移 + 方位偏移
  • 叠加顺序：全局层 → 角色层 → 方位层，依次叠加
  • 查找逻辑：引擎先查找全局偏移，再查找对应角色的偏移，最后查找对应方位的偏移，不存在的层级视为 0
  • 偏移方向：正值向右/向上偏移，负值向左/向下偏移
  • 作用范围：只影响表情层（face），不影响身体立绘（dress）
  • 配置可选：所有配置都是可选的，不存在的配置项视为 0
  • 预览屏蔽：此配置在可视化预览窗口中会被屏蔽，不影响预览效果

计算示例：

假设配置如下：

window.CHAR_DIFF_OFFSET = {
    "全局": { x: 1, y: 0.5 },
    "角色A": { "正面": { x: -0.5, y: 0 } }
};

当渲染「角色A 正面」时：

  • 全局偏移：x=1, y=0.5
  • 角色偏移：角色A 存在，但未配置「正面」方位，故方位偏移为 0
  • 最终偏移：x = 1 + 0 + 0 = 1，y = 0.5 + 0 + 0 = 0.5

当渲染「角色A 侧面」时（未配置）：

  • 全局偏移：x=1, y=0.5
  • 角色偏移：角色A 未配置「侧面」方位，故为 0
  • 最终偏移：x = 1 + 0 + 0 = 1，y = 0.5 + 0 + 0 = 0.5

实际应用：

引擎内部计算逻辑如下：

// 获取三层偏移
const globalOffset = CHAR_DIFF_OFFSET["全局"] || {};
const charOffset = CHAR_DIFF_OFFSET[charName] || {};
const orientOffset = charOffset[orientation] || {};

// 计算最终偏移
const finalOffsetX = (globalOffset.x || 0) + (charOffset.x || 0) + (orientOffset.x || 0);
const finalOffsetY = (globalOffset.y || 0) + (charOffset.y || 0) + (orientOffset.y || 0);

// 应用到表情位置
facePosition.x += finalOffsetX;
facePosition.y += finalOffsetY;
注意：CHAR_DIFF_OFFSET 的偏移值是叠加在 CHAR_DIFF_CONFIG 中配置的 face 坐标之上的，不会影响 dress 的位置。

三、坐标系与位置控制

3.1 配置中的 x / y / z 参数

每个 dress 和 face 对象都有三个坐标参数，使用屏幕百分比而非像素：

参数含义单位说明
x水平位置%距左边缘的屏幕宽度百分比（0=左边缘, 50=屏幕中心, 100=右边缘）。引擎以 translateX(-50%) 居中，即图片水平中心对齐到 x% 处
y垂直位置%距 #character-container（屏幕底部75%区域）底部的百分比。0=容器底部, 正值=向上移动, 负值=向下（图片底部延伸出屏幕）
z基准大小%控制图像高度占容器高度的百分比。0=图像高度100%容器, -20=80%, 30=130%, -91.5=8.5%。这是控制立绘整体大小的主控制器

坐标对照表（参考分辨率 1920×1080，容器 75vh=810px）：

  • X轴：100px = 5.21%，300px = 15.63%，650px = 33.85%
  • Y轴：100px = 12.35%，300px = 37.04%，528px = 65.19%
注意：
  • dress.x/dress.y：控制整个立绘容器的位置，作为基准坐标
  • face.x/face.y：使用与 dress 相同的坐标系，引擎自动计算差值作为层内偏移
  • face.z：控制表情相对于容器的基准大小百分比

四、可视化调试窗口

引擎提供了可视化立绘预览窗口，方便你在编写剧本时实时查看立绘效果。

  • 快捷键：F9
  • 可用版本：仅在 Shiori_debug.exe 中可用，位于「调试」菜单中。
  • 功能说明：
    • 窗口会自动解析 CHAR_DIFF_CONFIG 中配置的所有立绘差分。
    • 仅展示新系统（高级）立绘，旧版简单立绘不做展示。
    • 复制出来的差分为纯 ID（方位 + 服装 + 表情），不包含修饰词——请在剧本中自行补充位置、动作等修饰词。
    • CHAR_DIFF_OFFSET 在预览窗口中被屏蔽，确保预览效果不受剧本偏移配置影响。
    • 立绘容器占满全部高度（无对话框区域），提供更完整的预览空间。

五、进阶技巧与注意事项

5.1 资源文件组织建议

推荐的目录结构：

assets/
└── chars/
    ├── 角色A/
    │   ├── charA_front_dress1.png
    │   ├── charA_front_face1.png
    │   └── charA_side_dress1.png
    ├── 角色B/
    │   ├── charB_front_dress1.png
    │   └── charB_front_face1.png
    └── ...

然后在 CHAR_DIFF_CONFIG 中只需要写文件名，在 CHAR_PATH_MAP 中配置路径前缀即可。这样更换资源目录时只需修改一处配置。

5.2 配置变量命名规范

所有配置变量必须使用 window. 前缀定义，否则引擎无法访问：

// 正确
window.CHAR_DIFF_CONFIG = { ... };
window.CHAR_PATH_MAP = { ... };
window.CHAR_DIFF_OFFSET = { ... };

// 错误（引擎无法访问）
const CHAR_DIFF_CONFIG = { ... };

5.3 新旧系统混用

高级立绘系统与旧版简单立绘系统完全兼容，可以在同一条指令中混合使用：

chars="[角色A 中 正面 服装1 表情101], [角色B 右 lh12345]"

引擎会自动检测指令格式并使用对应的渲染方式。

5.4 注意事项

  • face 的位置计算：face 使用与 dress 相同的坐标系，引擎会自动计算两者差值作为层内偏移
  • CHAR_DIFF_OFFSET 仅影响表情：此配置仅对表情层生效，不会影响身体立绘的位置
  • 可视化窗口屏蔽偏移：在可视化预览窗口中，CHAR_DIFF_OFFSET 会被置空，确保预览效果不受剧本偏移配置影响
  • 修饰词屏蔽：百分比缩放指令和包含缩放效果的动画已在新版立绘中自动屏蔽，详见第六章
  • 旧系统文档：关于位置修饰词、动作指令、渐变动画、亮度控制等通用功能，请参考 illustration.md 文档

六、新版立绘屏蔽功能

新版立绘系统对部分修饰词进行了自动屏蔽处理。以下是完整的屏蔽清单和行为说明：

6.1 屏蔽原理

所有屏蔽操作在指令解析阶段（parseCharDiffModifiers 函数）完成。被屏蔽的修饰词会被安全忽略——不会导致报错、脚本中断或异常行为，立绘渲染完全不受影响。控制台会输出 [新版屏蔽] 日志便于调试。

兼容性保障：屏蔽功能仅对新版立绘系统生效。旧版简单立绘系统中的所有修饰词保持不变，后退、前进、吓一跳动作和百分比缩放均正常可用。

6.2 百分比缩放指令

所有百分比形式的缩放指令在渲染过程中被忽略，立绘始终保持原始大小（scale=1）。

指令示例旧系统效果新版系统效果
[10%]放大到 110%屏蔽 — 保持 100%
[-20%]缩小到 80%屏蔽 — 保持 100%
[30%]放大到 130%屏蔽 — 保持 100%
[0%]保持 100%屏蔽 — 保持 100%

如需在新版立绘中调整大小，请使用 CHAR_DIFF_CONFIG 中的 z 参数（基准大小），或通过 CHAR_PATH_MAP 配置不同尺寸的资源文件。

6.3 包含缩放效果的动画

以下三个涉及 scale 变换的动画在解析时被过滤，actionType 保持 null，动画不会被执行：

指令中文/英文屏蔽原因
后退 / retreatretreat涉及缩放 -10%
前进 / forwardforward涉及缩放 +10%
吓一跳 / scarescare涉及缩放 ±7% 循环

6.4 不受影响的动画

以下不包含缩放效果的动画在新版立绘中正常可用：

中文英文效果
发抖shakeX轴左右偏移
持续发抖cshake持续左右抖动
结束发抖sshake停止持续发抖
点头nod上移→下移→回归
左倒ltilt逆时针旋转90度
右倒rtilt顺时针旋转90度
站立stand重置旋转角度
得意proud单次得意摇摆
持续得意cproud持续得意摇摆
结束得意sproud停止持续得意

6.5 写入被屏蔽内容的行为

如果你在剧本中编写了被屏蔽的修饰词（例如 [角色A 中 正面 服装1 表情101 10% 后退]），新版立绘系统会：

  1. 安全过滤：百分比 10% 会被忽略，立绘保持原始大小
  2. 跳过动画：后退 动作不会被执行，立绘不产生位置/缩放变化
  3. 正常渲染：其余修饰词（位置 中、方位 正面、服装 服装1、表情 表情101）照常生效
  4. 控制台日志：在浏览器开发者工具的控制台中会看到类似以下的日志输出：
[CharDiffParser] [新版屏蔽] 百分比缩放指令已过滤: '10%' -> 强制保持 100% (scale=1)
[CharDiffParser] [新版屏蔽] 包含缩放效果的动画已过滤: '后退' (retreat)
无需修改现有剧本：如果你或他人习惯在指令中添加缩放修饰词，无需逐一排查。所有被屏蔽的修饰词均会被引擎自动安全处理，不会导致渲染异常或脚本错误。
